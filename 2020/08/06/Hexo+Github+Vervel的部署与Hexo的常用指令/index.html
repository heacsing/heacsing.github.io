Hexo+Github+Vervel的部署与Hexo的常用指令

环境与准备

  • Google node.js官网下载最新版node.js，测试是否安装成功：
  • Google Git 下载并安装，注册一个GitHub账号
  • 通过npm包管理器下载hexo：

    Hexo部署

  • 新建目标文件夹并初始化：在父文件夹下执行
  • 在该目录下执行，更新插件，部署到GitHub还需要插件hexo-deployer-git
  • 进入自己的github，新建一个仓库命名为{{account name}}.github.io，并配置ssh密钥——具体操作google it
  • Github每个账号可以有一个个人主页，但每个repo都可以有一个项目主页，通过设置repo的git pages，即可通过URL{{account name}}.github.io/{{repo name}}来访问
  • 修改根目录下配置文件_config.yml
  • 通过hexo命令部署

    Hexo的一些常用指令

  1. hexo clean：清理cache
  2. hexo g/generate：生成静态网页
  3. hexo s/server：基于node.js的本地服务
  4. hexo d/deploy：将工程部署到远端

    Hexo的美化，以及架构理解

  5. 主题/theme
    • 在官方网站上可以找到很多好看的主题，一般可以通过npm和git两种方式安装
      1. 在根目录themes文件夹下，
      2. 直接
    • 之后修改根目录下配置文件_config.yml
  6. live2D
    • 低级形态：使用hexo-live2d-helper部署
    • 高级形态：使用live2d-widget部署
    • 注意前者在根目录下部署，后者在主题文件下部署
    • 一些模型库，library, demo
  7. Hexo渲染原理探索
    • 在本地运行hexo g时，首先读取根目录下_config.yml设置，再利用选定的主题文件进行渲染：
    • 在themes/{{theme name}}/layout文件夹下可以找到进行渲染的ejs文件与_partial文件夹，而css和js文件可在同父目录下source文件夹里找到，基于此，可以进行主题的再修改(mo gai)
    • 图片等资源基本采用图床引用

      vercel部署

  • 注册一个vercel账号，这里有一个坑就是他们把@qq.comban掉了，所以qq邮箱是无法注册的，也无法通过qq邮箱注册的github或者bitbucket账号登录，虽然GitHub是qq注册的但幸好bitbucket是gmail
  • 但注册好后，就可以将账号与GitHub账号正常关联在一起，从GitHub上import项目即可，这里需要注意的一个坑就是，引入项目时会有一个FRAMEWORK PRESET(框架预设值)的选项，不要选择Hexo，other即可，否则会导致部署的项目404
